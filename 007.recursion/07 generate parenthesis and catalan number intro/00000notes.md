Given an integer n.Generate all possible combinations of well-formed parentheses of length 2 x N.


Examples:
Input : n = 3

Output : [ "((()))" , "(()())" , "(())()" , "()(())" , "()()()" ]

Input : 2

Output : [ "(())" , "()()" ]

Input : 1

Output:
()
Constraints:
1 <= n <= 8
## Right code

```cpp

class Solution {
    void generate (int n ,int open,int close,vector<string>&res,string &str){
            if(open+close==2*n ) {
                res.push_back(str);
                return;
            }
            if(open<n){
                 str+="(";
                generate(n,open+1,close,res,str);
                str.pop_back();
            }
            
            if(close<open) {
                str+=")";
                generate(n,open,close+1,res,str);
                str.pop_back();
            }
    }
public:
    vector<string> generateParenthesis(int n) {
        vector<string> res;
        string str="";
        generate(n,0,0,res,str);
        return res;
    }
};
```
The time complexity (TC) of this recursive backtracking solution for generating all valid combinations of $n$ pairs of parentheses is derived from two main factors: the number of solutions and the time taken to build them.

### Time Complexity Analysis

1.  **Number of Valid Solutions ($C_n$):**
    The total number of valid strings generated is exactly the $n$-th **Catalan number**, $C_n$.
    $$
    C_n = \frac{1}{n+1} \binom{2n}{n}
    $$
    Using Stirling's approximation, the asymptotic growth rate for Catalan numbers is:
    $$
    C_n \approx O\left(\frac{4^n}{n^{3/2}}\right)
    $$

2.  **Time to Generate and Store:**
    The recursion tree involves two choices at each step (`str += "("` or `str += ")"`), and the depth of the recursion is $2n$ (the length of the final string).
    * The length of every generated string is **$2n$**.
    * The operation `res.push_back(str)` takes **$O(n)$** time because it involves copying the entire string of length $2n$.

The overall time complexity is:
$$\text{TC} = \text{(Number of Solutions)} \times \text{(Time to Store Each Solution)}$$

$$
TC = O(C_n) \cdot O(n) = O(n \cdot C_n)
$$

### Conclusion

The time complexity of the provided code is **$O(n \cdot C_n)$**, which, using the approximation for $C_n$, is commonly expressed as:

$$
O\left(\frac{4^n}{\sqrt{n}}\right)
$$
A looser, exponential upper bound is $O(4^n)$.

The number of valid combinations of $n$ pairs of parentheses is exactly the $n$-th **Catalan number ($C_n$)**. This is because the core constraint of the problem—that the count of open parentheses must always be greater than or equal to the count of closed parentheses in any prefix—is the precise condition that defines the Catalan sequence.

The formula for the $n$-th Catalan number is:
$$
C_n = \frac{1}{n+1} \binom{2n}{n}
$$

# Catalan Number Formulas

The **Catalan number** ($C_n$) has several equivalent formulas.
Here are the most common and useful ones:

---

## ✅ 1. Closed-form (Most Popular)

$$
C_n = \frac{1}{n+1} \binom{2n}{n}
$$

Expanded:

$$
C_n = \frac{(2n)!}{(n+1)!\,n!}
$$

---

## ✅ 2. Recursive Formula

$$
C_0 = 1
$$

$$
C_n = \sum_{i=0}^{n-1} C_i \, C_{n-1-i}
$$

**Example:**

$$
C_3 = C_0C_2 + C_1C_1 + C_2C_0 = 1\cdot2 + 1\cdot1 + 2\cdot1 = 5
$$

---

## ✅ 3. Multiplicative Formula (Efficient for Coding)

This avoids factorials and is great for iterative computation:

$$
C_n = \frac{4n - 2}{n + 1} \cdot C_{n-1}
$$

or

$$
C_n = C_{n-1} \cdot \frac{2(2n - 1)}{n + 1}
$$
