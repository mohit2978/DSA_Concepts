# Count and say

The count-and-say sequence is a sequence of digit strings defined by the following recursive formula:

countAndSay(1) = "1"

For $n > 1$, countAndSay($n$) is generated by describing countAndSay($n-1$) in terms of the frequency and value of consecutive identical digits.

For example:

"1" is read as "one 1" or "11".

"11" is read as "two 1s" or "21".

"21" is read as "one 2, then one 1" or "1211".

"1211" is read as "one 1, one 2, then two 1s" or "111221".

"111221" is read as "three 1s, two 2s, then one 1" or "312211".

---

Given a positive integer $n$, return the $n^{\text{th}}$ term of the count-and-say sequence.

----

Examples:
Input: n = 4



Output: "1211"



Explanation:

countAndSay(1) = "1"

countAndSay(2) is described as "one 1" = "11"

countAndSay(3) is described as "two 1s" = "21"

countAndSay(4) is described as "one 2, then one 1" = "1211"

---

Input: n = 1



Output: "1"



Explanation:

This is the base case where countAndSay(1) is "1".

---

Constraints:
1 <= n <= 30

---
>Note:Simple recursion solution


```cpp
class Solution {
    string helper(int n){
        if(n==1) return "1";

        string tres=helper(n-1);
        string s="";
        for(int i=0;i<tres.size();){
            char ch=tres[i];
            int cnt=0;
            while(i<tres.size() && ch==tres[i]){
                i++;
                cnt++;
            }
            s+=to_string(cnt);
            s+=ch;
        }
        return s;

    }
  public:
    string countAndSay(int n) {
        return helper(n);
    }   
};
```

Time Complexity:The time complexity is approximately O(len(result) * n), where len(result) is the length of the string generated in the previous call, and n is the input number because helper function calls itself n times.

Space Complexity:The space complexity is O(n) due to the recursive call stack and string concatenation. Specifically it depends on the depth of recursion, which is 'n', and the size of the strings generated at each step.
