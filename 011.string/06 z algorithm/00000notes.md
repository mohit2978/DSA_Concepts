# Z-algorithm

The **Z-Algorithm** is an efficient linear-time string matching algorithm used to find all occurrences of a **pattern** $P$ within a **text** $T$. It achieves $O(N+M)$ time complexity, where $N$ is the length of $Text$ and $M$ is the length of $Pattern$.

The core idea is to compute a **Z-array** for a combined string $S$, which is formed by concatenating the pattern, a unique separator, and the text:
$$S = P + '\$' + T$$
where '$\$$' is a character that is guaranteed not to appear in $P$ or $T$.

## 📜 Z-Array Definition
The **Z-array**, $Z$, is an auxiliary array of the same length as $S$, which is formed by concatenating the pattern, a unique separator, and the text:
$$S = P + '\$' + T$$

 For each index $i > 0$ in $S$, $Z[i]$ stores the length of the **longest substring** starting at $S[i]$ that is also a **prefix** of $S$.

## Naive approach

The following dry run demonstrates the **naive $O(N^2)$ approach** to computing the Z-array, using the pattern and text .

In the naive approach, for every index $i$, we explicitly compare the substring $S[i \dots]$ with the prefix $S[0 \dots]$ character by character until a mismatch is found or the end of the string is reached.

### Input Parameters

* **Pattern** $P$: `"aab"` (Length $M=3$)
* **Text** $T$: `"xaabaaab"` (Length $N=8$)
* **Combined String** $S = P + '\$' + T$: `"aab$xaabaaab"` (Length $L_{S}=12$)
* **Text Start Index**: $i_{start} = M+1 = 4$

$$Z = [0, 1, 0, 0, 0, 3, 1, 0, 2, 3, 1, 0]$$

### Step 3: Identify Pattern Matches

The pattern is found in the text $T$ at index $i - 4$ when $Z[i]$ equals the pattern length $M=3$ and $i \ge 4$.

| $i$ | $Z[i]$ | Match? | Text Index $i - 4$ |
| :-: | :-: | :---: | :---: |
| 5 | 3 | **YES** | $5 - 4 = 1$ |
| 9 | 3 | **YES** | $9 - 4 = 5$ |

The pattern is found at text indices **1** and **5**.

```cpp
#include <bits/stdc++.h>
using namespace std;

class Solution {
private:
    vector<int> computeZarray(string s) {
        int n = s.size();
        
        vector<int> Z(n, 0);
        
        for(int i=1; i < n; i++) {
            while(i + Z[i] < n && s[i + Z[i]] == s[Z[i]]) 
                    Z[i]++;
        }
        
        return Z;
    }
    
public:
    vector<int> search(string text, string pattern)   {
        string s = pattern + '$' + text;
        
        vector<int> Z = computeZarray(s);
        
        int n = text.size(), m = pattern.size();
        
        vector<int> ans;
        
        for(int i = m+1; i < s.size(); i++) {
            if(Z[i] == m) ans.push_back(i - (m + 1));
            //i-(m+1) to get original index of text
        }
        
        return ans;
    }
};

int main() {
    string text = "xyzabxyzabxyz";
    string pattern = "xyz";
    
    Solution sol;
    
    vector<int> ans = sol.search(text, pattern);
    
    for(auto ind : ans) cout << ind << " ";
    return 0;
}
```

## Optimal approach

The following dry run applies to the C++ code provided, which implements the $O(N)$, where N is the length of the combined string S.

Computing the Z-array using modified approach takes O(N) time as each character of the string is processed at most once. And finding the matches requires iterating on the Z-array taking O(N) time.

 Z-Algorithm using the optimized two-pointer approach ($L$ and $R$ for the Z-box).

### Input Parameters

* **Pattern** $P$: `"aab"` (Length $M=3$)
* **Text** $T$: `"xaabaaab"` (Length $N=8$)
* **Combined String** $S = P + '\$' + T$: `"aab$xaabaaab"` (Length $L_{S}=12$)


## The Z-Box $[L, R]$

The Z-box is the pair of indices $[L, R]$ that represents the current substring $S[L \dots R]$ which is the longest prefix-matching segment found **that extends farthest to the right.**

* $S[L \dots R] = S[0 \dots R-L]$ (The substring matches the prefix).
* $R$ is the rightmost endpoint of any previous Z-match.

The algorithm iterates through the string with index $i$, calculating $Z[i]$. The pointers $L$ and $R$ are only updated at the very end of processing $i$, **if** the match starting at $i$ extends beyond the current $R$.



### Case 1: $i$ is Outside the Current Z-Box ($i > R$)

When the current index $i$ is greater than $R$, it means $i$ falls outside the current known Z-box. We have no prior information to leverage, so we must calculate $Z[i]$ naively.

**Update Logic:**

1.  **Reset $L$ and $R$:** Since $i$ is outside the current box, we reset the Z-box to start at $i$: $L \leftarrow i$ and $R \leftarrow i$.
2.  **Explicit Comparison (Naive):** We compare characters $S[R]$ with $S[R-L]$ (which is $S[0]$) and continue incrementing $R$ as long as they match and $R$ is within the string bounds.
    * `while (R < n && S[R-L] == S[R]) { R++; }`
3.  **Set $Z[i]$:** $Z[i] \leftarrow R - L$ (The length of the new match).
4.  **Final $R$ Adjustment:** $R$ will be pointing one character *past* the end of the match (either due to a mismatch or end-of-string). We must backtrack $R$: $R \leftarrow R - 1$.
    * The new Z-box is now $[L, R]$.

***$L$ and $R$ are updated because a new, longer Z-box starting at $i$ was established.***

### Case 2: $i$ is Inside the Current Z-Box ($i \le R$)

When $i$ is inside the Z-box $[L, R]$, the segment $S[i \dots R]$ must match the corresponding prefix segment $S[i-L \dots R-L]$ because $S[L \dots R]$ matches $S[0 \dots R-L]$.

This means the value of $Z[i]$ is related to the previously calculated $Z$ value at the corresponding relative index, $k = i - L$.

We determine the maximum possible length of the match starting at $i$ using two quantities:
1.  **$Z[k]$**: The match length at the corresponding position $k$ in the prefix.
2.  **$R - i + 1$**: The remaining length of the current Z-box starting from $i$.

#### Case 2a: Contained Match ($Z[k] < R - i + 1$)

* The match length $Z[k]$ is shorter than the remaining length of the Z-box.
* The mismatch that stopped $Z[k]$ occurs *inside* $S[L \dots R]$.
* Because the segment $S[L \dots R]$ matches the prefix $S[0 \dots R-L]$, the mismatch must also occur when matching $S[i \dots]$ with $S[0 \dots]$.
* **$L$ and $R$ are not updated.** $Z[i]$ is simply copied from $Z[k]$: $Z[i] \leftarrow Z[k]$.

#### Case 2b: Extending Match ($Z[k] \ge R - i + 1$)

* The match length $Z[k]$ is equal to or longer than the remaining length of the Z-box.
* We know $S[i \dots R]$ matches the prefix, but the match might extend *beyond* $R$.
* **$L$ is updated:** We set $L \leftarrow i$.
* **$R$ is potentially incremented:** We explicitly compare characters starting from $S[R+1]$ and $S[R-L+1]$ to extend the match as far as possible. $R$ increases during this explicit comparison.
    * `while (R < n && S[R-L] == S[R]) { R++; }`
* **Set $Z[i]$:** $Z[i] \leftarrow R - L$ (The length of the new match).
* **Final $R$ Adjustment:** $R \leftarrow R - 1$.

***$L$ and $R$ are updated only if we successfully extend the match past the previous $R$.***

### Summary of $L$ and $R$ Movement

| Condition | Action on $L$ and $R$ | Reason |
| :--- | :--- | :--- |
| **$i > R$** (Outside Z-Box) | $L \leftarrow i$, $R \leftarrow \text{New End of Match}$ | No previous information; must find a new Z-box from scratch. |
| **$i \le R$ and $Z[k] < R-i+1$** (Contained Match) | **$L$ and $R$ are unchanged.** | The longest match starting at $i$ is fully contained in the current Z-box. |
| **$i \le R$ and $Z[k] \ge R-i+1$** (Extending Match) | $L \leftarrow i$, $R \leftarrow \text{New End of Match}$ | A new, potentially longer Z-box starts at $i$. $R$ moves forward if the match can be extended. |



```cpp
#include <iostream>
#include <vector>
#include <string>
#include <algorithm> // Required for std::min

using namespace std;

// Function to compute the Z-array for a given string S in O(N) time.
// This function efficiently utilizes the Z-box [L, R] to avoid redundant comparisons.
vector<int> calculateZArray(const string& s) {
    int n = s.length();
    vector<int> z(n, 0);

    // L and R define the Z-box: the segment S[L...R] which is the rightmost 
    // prefix-matching segment found so far. R is the endpoint.
    int L = 0, R = 0;

    // Start iteration from i=1 since Z[0] is always 0 (or ignored).
    for (int i = 1; i < n; ++i) {
        
        // --- Core Optimization Logic (Case 2: i <= R) ---
        // If i is inside the current Z-box, we can initialize Z[i] based on Z[k].
        if (i <= R) {
            // k is the index symmetric to i with respect to L (k = i - L).
            //k is Left of i 
            int k = i - L;
            
            // remaining_length is R - i + 1. This is the guaranteed match length.
            int remaining_length = R - i + 1;
            //are calculating the length of the segment S[i ... R]. This segment represents 
            //the portion of the current Z-box that starts at the index being 
            //processed, i.

            /*
            S[L ... R] is a copy of S[0 ... R-L], and i is within L and R, the entire segment S[i ... R]$ must match the corresponding segment in the prefix, S[i-L ... R-L].
            */
            
            // Case 2a: Z[k] is strictly less than the remaining length.
            // The match is fully contained within the current Z-box.
            // Z[i] is safely copied from Z[k]. L and R remain unchanged.
            z[i] = min(remaining_length, z[k]);
        } 
        
        // --- Explicit Comparison and Z-Box Update (Case 1: i > R or Case 2b: Extend) ---
        // We always try to extend the match starting at i:
        // 1. If i was outside the Z-box (i > R).
        // 2. If Z[i] was initialized to the remaining length (Case 2b), meaning 
        //    the match might extend past R.
        
        // Note: The C++ standard idiom for the Z-algorithm often combines Case 1 
        // and Case 2b logic here, as both require explicit comparisons to advance R.
        while (i + z[i] < n && s[z[i]] == s[i + z[i]]) {
            z[i]++;
        }
        
        // --- Update L and R Pointers ---
        // If the match starting at i extends beyond the current R, update the Z-box.
        if (i + z[i] - 1 > R) {
            L = i;
            R = i + z[i] - 1; // R points to the rightmost end of the new match.
        }
    }
    
    return z;
}

// Function to find all occurrences of pattern P in text T using the Z-Algorithm.
vector<int> zAlgorithmSearch(const string& text, const string& pattern) {
    // 1. Construct the combined string S = P + '$' + T.
    string combined_string = pattern + "$" + text;
    int pattern_len = pattern.length();
    
    // 2. Compute the Z-array for the combined string.
    vector<int> z_array = calculateZArray(combined_string);

    vector<int> match_indices;
    
    // 3. Scan the Z-array starting from the index after the separator.
    // Text starts at index i = pattern_len + 1.
    for (int i = pattern_len + 1; i < combined_string.length(); ++i) {
        // A match is found if Z[i] equals the pattern length.
        if (z_array[i] == pattern_len) {
            // The match index in the original text is:
            // i (start of match in combined string) - (pattern_len + 1) (offset of text start)
            int text_index = i - (pattern_len + 1);
            match_indices.push_back(text_index);
        }
    }

    return match_indices;
}

int main() {
    // Example usage
    string pattern = "aab";
    string text = "xaabaaab";

    cout << "--- Optimized Z-Algorithm Pattern Search ---" << endl;
    cout << "Pattern P: \"" << pattern << "\"" << endl;
    cout << "Text T:    \"" << text << "\"" << endl << endl;

    // Output the computed Z-array and the results
    string combined = pattern + "$" + text;
    vector<int> z_array = calculateZArray(combined);

    cout << "Combined String: " << combined << endl;
    cout << "Indices (0-11):  012345678901" << endl;
    cout << "Z-Array:         ";
    for (int val : z_array) {
        cout << val;
    }
    cout << endl << endl;

    vector<int> matches = zAlgorithmSearch(text, pattern);

    if (matches.empty()) {
        cout << "Pattern not found in the text." << endl;
    } else {
        cout << "Pattern found at the following 0-based index(es) in the text T:" << endl;
        for (int index : matches) {
            cout << "-> Index " << index << endl;
        }
    }

    // Another example
    cout << "\n--- Another Example ---" << endl;
    pattern = "ABA";
    text = "ABAABA";
    cout << "Pattern P: \"" << pattern << "\"" << endl;
    cout << "Text T:    \"" << text << "\"" << endl;
    matches = zAlgorithmSearch(text, pattern);

    cout << "Pattern found at the following 0-based index(es) in the text T:" << endl;
    for (int index : matches) {
        cout << "-> Index " << index << endl;
    }

    return 0;
}
```

The $L$ and $R$ pointers in the optimized $O(N)$ Z-Algorithm define the boundaries of the **Z-box** (sometimes called the "Z-window" or "Z-block").

The Z-box $[L, R]$ is the single, most crucial concept that enables the Z-Algorithm to achieve linear time complexity.

Here is what the $L$ and $R$ pointers denote:

---

## The Z-Box: The Rightmost Prefix Match

The Z-box $[L, R]$ defines the segment of the string $S[L \dots R]$ that meets two conditions simultaneously:

1.  **It is a prefix match:** The substring $S[L \dots R]$ is identical to the corresponding prefix of the entire string $S[0 \dots R-L]$.
    $$S[L \dots R] = S[0 \dots R-L]$$
2.  **It is the Rightmost:** Of all prefix matches found so far (from index $i=1$ up to the current position), the Z-box $[L, R]$ is the one whose end point, $\mathbf{R}$, is the **largest** (i.e., extends farthest to the right).

>Note:In plain English, the Z-box $[L, R]$ is a segment within the string that, when you slide it back to the beginning of the string (index 0), it perfectly aligns with the starting characters of the string.

### 1. The $L$ Pointer (Left Boundary)

$L$ stands for **Left**. It denotes the **starting index** of the prefix-matching substring that currently defines the Z-box.

* $L$ is simply the index $i$ at which the most rightward-extending prefix match was found.
* The value of $Z[L]$ is always equal to the length of the Z-box ($R - L + 1$).

### 2. The $R$ Pointer (Right Boundary)

$R$ stands for **Right**. It denotes the **ending index** of the Z-box (the current rightmost prefix-matching segment).

* $R$ is always the largest index achieved by $i + Z[i] - 1$ so far in the iteration.
* The primary mechanism for the $O(N)$ speed is that $R$ only ever moves forward. Every time an explicit character comparison is performed (in the `while` loop), it contributes directly to advancing $R$. Since $R$ can only advance $N$ times (from $0$ to $N-1$), the total number of explicit comparisons is bounded by $O(N)$.

---

## How $L$ and $R$ are Used for Optimization

When calculating $Z[i]$, the algorithm checks the relationship between $i$ and $R$:

### If $i > R$ (Outside the Z-Box)

The Z-box provides no information about $Z[i]$. We must start a naive comparison to find $Z[i]$. If a new match is found that extends farther than the old $R$, both $L$ and $R$ are updated to reflect the boundaries of this new, longer Z-box.

### If $i \le R$ (Inside the Z-Box)

This is the optimization step. Because $S[L \dots R]$ is a prefix match, the segment $S[i \dots R]$ is guaranteed to match the segment $S[i-L \dots R-L]$.

1.  **Symmetry:** We look up $Z[k]$ where $k = i - L$. This tells us how long the match is at the symmetric position $k$ in the prefix.
2.  **Guaranteed Match:** We know $Z[i]$ is at least $\min(Z[k], R - i + 1)$. This minimum value is calculated without any character comparisons, which is the time saver.
3.  **Update Rule:** We only perform explicit character comparisons (advancing $R$) if $Z[k]$ suggests the match might extend *beyond* the current $R$. If it does, $L$ and $R$ are updated. If it does not, $L$ and $R$ remain unchanged.

### The Principle of the Guaranteed Match

The core guarantee comes from the definition of the Z-box $[L, R]$:

1.  **Prefix Property:** By definition of $L$ and $R$, the segment $S[L \dots R]$ **must** match the prefix $S[0 \dots R-L]$.
    $$\mathbf{S[4 \dots 7]} = \text{"aaba"} \quad \text{matches} \quad \mathbf{S[0 \dots 3]} = \text{"aaba"}$$

2.  **Symmetry:** Now consider the index $i=5$, which is inside the box ($5 \le 7$). We are interested in the substring $S[i \dots R]$, which is $S[5 \dots 7] = \text{"aba"}$.

3.  **The Guarantee:** Because $S[L \dots R]$ matches the prefix i.e. $S[0 \dots R-L]$., any segment within $S[L \dots R]$ must match its corresponding segment in the prefix $S[0 \dots R-L]$.



    * The segment we are checking is $S[i \dots R] = S[5 \dots 7]$.
    * The length of this segment is $R - i + 1 = 7 - 5 + 1 = 3$.
    * The corresponding segment in the prefix starts at the symmetric index: $k = i - L = 5 - 4 = 1$.
    * The corresponding segment in the prefix is $S[k \dots k + (R-i)]$, which is **$S[1 \dots 3]$** ($\text{"aba"}$).

Since $S[4 \dots 7]$ matches $S[0 \dots 3]$, it is **mathematically guaranteed** that the internal segment $S[5 \dots 7]$ matches the internal segment $S[1 \dots 3]$.

$$\underbrace{\mathbf{S[5 \dots 7]}}_{\text{The segment starting at } i=5} = \text{"aba"} \quad \text{must match} \quad \underbrace{\mathbf{S[1 \dots 3]}}_{\text{The segment starting at } k=1}$$

### How the Z-Algorithm Uses This

Because of this guaranteed match, we can skip direct character comparisons for the entire segment $S[i \dots R]$. Instead, we look at the value $Z[k]$:

1.  **Find $Z[k]$:** We look up $Z[1]$. Let's assume $Z[1]=1$ (as $S[1..]$ only matches 'a').
2.  **Find Guaranteed Match Length:** The guaranteed match length is $R-i+1 = 3$.
3.  **The $\min$ Operation:** We take $\min(Z[1], 3) = \min(1, 3) = 1$. This means the length of the prefix match starting at $i=5$ is *at least* 1, and since $Z[1]$ stopped early at length 1, $Z[5]$ cannot possibly be longer than 1.

The ability to compute this minimum value using only array lookups, without any explicit character comparisons for the length $R-i+1$, is why the algorithm runs in linear time.

## Updation of L and R

This is a perfect request to summarize everything we've discussed about the Z-Algorithm's efficiency. The way $L$ and $R$ are updated is the secret to its $O(N)$ performance.

The update of $L$ and $R$ is governed by the single condition: **Did the match starting at $i$ extend past the old $R$ value?**

Here is a breakdown of how $L$ and $R$ are updated in each of the three logical cases.

---

## The $L$ and $R$ Update Logic

The pointers $L$ (Left) and $R$ (Right) define the Z-box, the most rightward-extending prefix match found so far.

The update always happens *after* $Z[i]$ has been fully calculated (after the `while` loop runs):

```cpp
if (i + z[i] - 1 > R) {
    L = i;
    R = i + z[i] - 1; // R is the new end-point of the match
}
```

### Case 1: $i > R$ (Outside the Z-Box)

This is the "Initialization" or "Reset" case. The current Z-box offers no information, so we have to calculate $Z[i]$ from scratch using explicit character comparisons.

| How $L, R$ Update | Logic | Dry Run Example (S="aabaaaba", $i=3$) |
| :---: | :--- | :--- |
| **$L \leftarrow i$** | $i$ is outside the old box, so we must start a new search from $i$. If a match is found, $i$ becomes the starting point of the new rightmost box. | $i=3$ is outside $R=1$. The match $S[3..5]$ is found. $L$ moves from $1$ to **$3$**. |
| **$R \leftarrow i + Z[i] - 1$** | The entire length of the match $Z[i]$ was just found by explicit comparison. Since $Z[i]>0$, the new end-point is guaranteed to be $\ge R_{old}$, so $R$ updates to the new end. | The match length is $Z[3]=3$. The new endpoint is $3+3-1=5$. $R$ moves from $1$ to **$5$**. |
| **Result:** | **$L$ and $R$ are always updated** if $Z[i] > 0$. | The new Z-box is **$[3, 5]$**. |


### Case 2a: $i \le R$ and $Z[k] < R-i+1$ (Contained Match)

This is the "Copy" case. $Z[i]$ is copied from the symmetric value $Z[k]$, and the mismatch is known to occur inside the old Z-box $[L, R]$.

| How $L, R$ Update | Logic | Dry Run Example (S="aabaaaba", $i=4$) |
| :---: | :--- | :--- |
| **$L$ remains $L$** | The match at $i$ is strictly shorter than the guarantee ($R-i+1$). It cannot extend past the current $R$. | $i=4$. Match end is $4+Z[4]-1 = 4+1-1=4$. |
| **$R$ remains $R$** | Since $i + Z[i] - 1$ (which is $4$) is not greater than $R_{old}$ (which is $5$), the $R$ pointer does not move. | $4 \not> 5$. $R$ remains **$5$**. |
| **Result:** | **$L$ and $R$ are NOT updated.** The optimization worked perfectly, and no Z-box boundary needed to change. | The Z-box remains **$[3, 5]$**. |


### Case 2b: $i \le R$ and $Z[k] \ge R-i+1$ (Extending Match)

This is the "Extend" case. The match starting at $i$ is guaranteed up to $R$ and might extend further. We must explicitly check characters past $R$.

| How $L, R$ Update | Logic | Dry Run Example (S="aabaaaba", $i=5$) |
| :---: | :--- | :--- |
| **$L \leftarrow i$** | The old Z-box has "expired" at $R$. We start the **new** Z-box from $i$. | $L$ moves from $3$ to **$5$**. |
| **$R \leftarrow i + Z[i] - 1$** | Since we performed explicit comparison past $R_{old}$ in the `while` loop (and succeeded), the new $R$ reflects the newly found farthest boundary. | The match length $Z[5]$ extends to 2. New endpoint is $5+2-1=6$. Since $6 > R_{old}=5$, $R$ moves from $5$ to **$6$**. |
| **Result:** | **$L$ and $R$ are always updated** because the point of this case is to find a potentially longer Z-box starting at $i$. | The new Z-box is **$[5, 6]$**. |
