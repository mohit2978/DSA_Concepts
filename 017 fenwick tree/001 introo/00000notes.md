# Fenwick Tree vs. Segment Tree

Both Fenwick Trees (Binary Indexed Trees) and Segment Trees are designed to solve "Range Query" problems, allowing you to update values and query ranges (sums, mins, maxes) in $O(\log N)$ time.

### Quick Comparison Table

| Feature | Fenwick Tree (BIT) | Segment Tree |
| :--- | :--- | :--- |
| **Best For** | Prefix sums and point updates | General range queries (min/max/GCD) |
| **Space Complexity** | $O(N)$ (minimal memory footprint) | $O(4N)$ (needs extra nodes) |
| **Update Time** | $O(\log N)$ (faster constant factors) | $O(\log N)$ |
| **Range Updates** | Complex to implement | Easy with **Lazy Propagation** |
| **Implementation** | Short, iterative, non-recursive | Longer, usually recursive |
| **Operations** | Only invertible (Sum, XOR, etc.) | Any associative (Min, Max, GCD, etc.) |

---

### 1. Fenwick Tree (Binary Indexed Tree)
The Fenwick Tree is a compact data structure that stores cumulative sums. It exploits the binary representation of an index to manage ranges.

* **How it works:** Each node at index `i` is responsible for a range of elements ending at `i`. The size of that range is determined by the "least significant bit" (LSB) of the index.
* **The "Invertible" Constraint:** To find the sum of range $[L, R]$, a BIT calculates `Sum(0 to R) - Sum(0 to L-1)`. This works for addition, but it **cannot** find the minimum or maximum in a range because you cannot "subtract" a minimum to isolate a sub-range.
* **Pros:** It is extremely memory-efficient and very fast in practice due to small constant factors. It is often the preferred choice for simple prefix sum problems.

### 2. Segment Tree
A Segment Tree is a full binary tree where each leaf represents an element, and each internal node represents the combined result of its children (a "segment").

* **How it works:** It recursively divides the array into halves until it reaches individual elements. Queries are answered by combining the pre-calculated results of these segments.
* **Versatility:** Unlike the BIT, it doesn't need to "subtract" ranges. To find the minimum in $[L, R]$, it simply identifies the specific segments that cover that range and takes the minimum of those values.
* **Lazy Propagation:** This is a major advantage. It allows you to update an entire range (e.g., "add 10 to every element from index 5 to 50") in $O(\log N)$ time. Fenwick Trees typically require $O(N \log N)$ or complex modifications for this.

---

### Which one should you use?

* **Use a Fenwick Tree if:** You only need **Range Sum Queries** and **Point Updates**, or if you are constrained by strict memory limits.
* **Use a Segment Tree if:** You need to handle **Range Minimum/Maximum Queries**, or if you need to perform efficient **Range Updates**.

```java
class NumArray {
    class fenwickTree{
        private int[] fenwick;
        fenwickTree(int[] arr){
            fenwick=new int[arr.length+1];
            for(int i=1;i<fenwick.length;i++){
                update(i,arr[i-1]);
            }
        }
        private int rMsB(int x){
            return x&-x;
        }
        int sum(int pos){
            int res=0;
            while(pos>0){
                res+=fenwick[pos];
                pos-=rMsB(pos);
            }
            return res;
        }
        void update(int pos,int del){
            while(pos<fenwick.length){
                fenwick[pos]+=del;
                pos+=rMsB(pos);
            }
        }
    }
    fenwickTree ft;
    int[] arr;
    public NumArray(int[] nums) {
        ft=new fenwickTree(nums);
        arr=nums;
    }
    
    public void update(int index, int val) {
        int del=val-arr[index];
        ft.update(index+1,del);
        arr[index]=val;
    }
    
    public int sumRange(int left, int right) {
        int lpos=left+1;
        int rpos=right+1;
        return ft.sum(rpos)-ft.sum(lpos-1);
    }
}

/**
 * Your NumArray object will be instantiated and called as such:
 * NumArray obj = new NumArray(nums);
 * obj.update(index,val);
 * int param_2 = obj.sumRange(left,right);
 */
```

### By segment tree

```cpp
class NumArray {
    vector<int>segtree;
    int n=0;
    int sz=0;

 void buildTree(vector<int>& nums,int s,int e,int i){

    if(s==e){
        segtree[i]=nums[s];
        return;
    }

    int mid=(s+e)/2;
    buildTree(nums,s,mid,2*i+1);
    buildTree(nums,mid+1,e,2*i+2);

    segtree[i]=segtree[2*i+1]+segtree[2*i+2];

 }

 void updateTree(int idx,int val,int s,int e,int i){

    if(s==e){
        segtree[i]=val;
        return;
    }
    int mid=(s+e)/2;
    if(idx<=mid) updateTree(idx,val,s,mid,2*i+1);
    else updateTree(idx,val,mid+1,e,2*i+2);

    segtree[i]=segtree[2*i+1]+segtree[2*i+2];
 }

int getSum(int l,int r,int s,int e,int i){

    if(r<s || e<l) return 0;

    if(l<=s && e<=r) return segtree[i];

    int mid=(s+e)/2;

    return getSum(l,r,s,mid,2*i+1)+getSum(l,r,mid+1,e,2*i+2);
}

public:
    NumArray(vector<int>& nums) {
        n=nums.size();
        sz=4*n;
        segtree.resize(sz);
        buildTree(nums,0,n-1,0);
    }
    
    void update(int index, int val) {

        updateTree( index, val,0,n-1,0);
        
    }
    
    int sumRange(int left, int right) {
        return getSum(left,right,0,n-1,0);
    }
};

/**
 * Your NumArray object will be instantiated and called as such:
 * NumArray* obj = new NumArray(nums);
 * obj->update(index,val);
 * int param_2 = obj->sumRange(left,right);
 */
 ```

### Fenwick Tree for Range XOR (C++)

In C++, we use the `^=` operator for XOR updates and `i & -i` to isolate the least significant bit.

#### Implementation

```cpp
#include <iostream>
#include <vector>

class FenwickTreeXOR {
private:
    std::vector<int> tree;
    int n;

public:
    FenwickTreeXOR(int size) : n(size) {
        tree.assign(n + 1, 0);
    }

    // Standard Point Update: XORs the tree at index i with val
    // To replace a value at index i, pass (old_value ^ new_value)
    void update(int i, int val) {
        for (; i <= n; i += i & -i) {
            tree[i] ^= val;
        }
    }

    // Returns prefix XOR sum from 1 to i
    int query(int i) {
        int res = 0;
        for (; i > 0; i -= i & -i) {
            res ^= tree[i];
        }
        return res;
    }

    // Returns XOR sum of range [l, r] (1-based)
    int queryRange(int l, int r) {
        return query(r) ^ query(l - 1);
    }
};

int main() {
    std::vector<int> arr = {3, 5, 1, 2, 7};
    int n = arr.size();
    FenwickTreeXOR ft(n);

    // Initial construction
    for (int i = 0; i < n; i++) {
        ft.update(i + 1, arr[i]);
    }

    // Example Query: XOR of arr[0] to arr[2] (indices 1 to 3)
    // 3 ^ 5 ^ 1 = 7
    std::cout << "XOR range [1, 3]: " << ft.queryRange(1, 3) << std::endl;

    // Point Update: Change arr[1] (which is 5) to 10
    // We XOR the existing value with the new value: (5 ^ 10)
    int oldVal = 5, newVal = 10;
    ft.update(2, oldVal ^ newVal);

    // New Query: 3 ^ 10 ^ 1 = 8
    std::cout << "New XOR range [1, 3]: " << ft.queryRange(1, 3) << std::endl;

    return 0;
}
```
